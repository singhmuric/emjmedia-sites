## Kontext
Fix B aus 1.8.1 wurde ausgesetzt, weil im Code kein Top-Scrim mit
weißem Gradient existierte. Cowork-Klärung ergibt: was als „weißer
Nebel" gesehen wurde, sind die Decken-LED-Streifen der Werkstatt
im Hero-Foto selbst. Lösung: dünner dunkler Scrim drüberlegen, der
die Lichter optisch abdämpft. Spiegelbild zum Subline-Scrim aus
1.8 §2.2.

## Implementation
- Datei: `sites/onepages/kfz-demo/styles/sections/03-hero.css`
- Position: direkt nach `.hero__media img { … }`, vor dem `.hero__scrim`-Block
- Regel:
  ```css
  .hero__media::before {
    content: "";
    position: absolute;
    inset: 0 0 auto 0;
    height: 14%;
    background: linear-gradient(
      180deg,
      rgba(11, 18, 32, 0.55) 0%,
      rgba(11, 18, 32, 0.25) 55%,
      rgba(11, 18, 32, 0)    100%
    );
    pointer-events: none;
    z-index: 1;
  }
  ```

## Stacking-Voraussetzung verifiziert
- `.hero__media` ist `position: absolute; inset: 0; z-index: -2` (seit 1.6 unverändert).
- `position: absolute` schafft genau wie `position: relative` einen
  Containing Block für `position: absolute`-Kinder. Funktional
  äquivalent für die `inset:0 0 auto 0`-Positionierung des `::before`.
- `z-index: 1` auf `::before` liegt im lokalen Stacking-Context von
  `.hero__media` über `<img>` (in-flow Layer 3) und über
  `.hero__scrim` (positioned, kein z-index, Layer 6).
- `.hero__inner`-Inhalte (Headline, Subline, CTAs, Trust-Pills)
  liegen außerhalb von `.hero__media`, im normal flow → unter z-index
  „auto" stehen über `.hero__media` (z-index: -2) und damit über
  beiden Scrims. Bleiben unverändert sichtbar.
