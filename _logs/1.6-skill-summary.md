## Fail-Safe-PE-Strategie (Constitution §12.5, CLAUDE.md §5)

Wurde bei Variant A / Session 1.4 zum Bug (5 Sektionen leer trotz Lighthouse 100/100/96/69). Strukturell verhindert in Variant B durch:

```css
/* Default: sichtbar, keine Animation */
[data-reveal], [data-reveal-stagger] > * { opacity: 1; transform: none; }

/* Hidden NUR wenn JS erfolgreich initialisiert */
html.js-reveal-ready [data-reveal],
html.js-reveal-ready [data-reveal-stagger] > * {
  opacity: 0; transform: translateY(12px);
  transition: opacity 0.9s var(--ease-out), transform 0.9s var(--ease-out);
}

@media (prefers-reduced-motion: reduce) {
  html.js-reveal-ready [data-reveal],
  html.js-reveal-ready [data-reveal-stagger] > * {
    opacity: 1; transform: none; transition: none;
  }
}
```

```js
if ('IntersectionObserver' in window
    && !matchMedia('(prefers-reduced-motion: reduce)').matches
    && document.querySelector('[data-reveal], [data-reveal-stagger]')) {
  document.documentElement.classList.add('js-reveal-ready');
  const io = new IntersectionObserver(...);
  document.querySelectorAll('[data-reveal], [data-reveal-stagger]').forEach(el => io.observe(el));
}
```

→ Wenn JS bricht, langsam lädt, blockiert oder deaktiviert: `html.js-reveal-ready` wird NIE gesetzt → CSS-Default greift → **Content bleibt sichtbar**.
