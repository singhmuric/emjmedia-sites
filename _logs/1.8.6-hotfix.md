# Hotfix 1.8.6 — CTA Border Geometrie symmetrisch

**Datum:** 2026-04-25
**Branch:** feat/1.8-fixes (auf 1.8.5 draufgesetzt)
**Bezug:** Cowork-Preview-Review #6

---

## Befund (vor Fix)
User: „Auf linker Vertikalen läuft der Hot-Spot außerhalb der Button-
Kontur, auf rechter Vertikalen innerhalb." Innen/Außen-Wechsel.

## Root-Cause
- CSS (`.cta-border-loop`) hatte `inset: -3px` → SVG-Element war 6 px breiter als der Button.
- JS setzte SVG-Attribute `width = bbox.width + 6` und rect bei `(1.5, 1.5)` mit `width = bbox + 6 - 3`.
- Konflikt zwischen viewBox-Rasterung und CSS-Element-Skalierung führte bei Sub-Pixel-Bbox-Coords (`ctaBox.x = 370.21`, `width = 199.71`) zu asymmetrischer rect-Position — linker Stroke landete „außen", rechter „innen".

## Fix

### CSS (`styles/sections/03-hero.css`)
- `.cta-border-loop`: `inset: 0` (vorher -3px), `width/height: 100%`, `overflow: visible`, `transform: translateZ(0)`.
- `.cta-border-loop rect`: `vector-effect: non-scaling-stroke` (Stroke-Breite bleibt 2.5 device-px unabhängig von viewBox-Skalierung).

### JS (`scripts.js` updateCtaBorder)
- SVG = Button-Größe (vorher +6 px Reserve)
- rect deckungsgleich mit viewBox: `x=0, y=0, width=w, height=h`
- `preserveAspectRatio: none` (viewBox skaliert linear in beide Achsen)
- Sub-Pixel-Snap mit dpr unverändert
- 4 Trigger (initial / load / fonts.ready / resize) unverändert
