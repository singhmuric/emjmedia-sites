## §2.2 Hero-Subline-Scrim — Diagnose + Verstärkung

- [x] **Schritt 1: Vorher-Screenshot + Diagnose-Befund**
  - Screenshot: `_logs/1.8-hero-before.png` (Viewport 1440×900)
  - Befund: `_logs/1.8-hero-before.json` — min-Kontrast 3.12:1 bei Sample (172, 569), BG rgb(135.7, 136.2, 138.7)
- [x] **Schritt 2: Root-Cause identifiziert + behoben**
  - **Root-Cause:** `.hero__scrim` hatte `z-index: -1` INNERHALB von `.hero__media` (selbst `z-index: -2`). Im lokalen Stacking-Context werden in-flow Children (das `<img>`) ÜBER negative-z-index positioned Children gestackt. Der Scrim lag damit hinter dem Foto und war seit 1.7 unsichtbar. Beweis: identische Pixel-Werte vorher und nach Scrim-Verstärkung von 0.72 auf 0.88 (vor dem z-index-Fix).
  - Fix: `z-index: -1` aus `.hero__scrim` entfernt → Scrim auto-positioned (Stacking-Layer 6) über img (Layer 3). Datei: `sites/onepages/kfz-demo/styles/sections/03-hero.css` Zeilen 22–60.
- [x] **Schritt 3: Scrim verstärkt (Option A)**
  - Gradient von `linear-gradient(100deg, 0.72 → 0 bei 72%)` auf `linear-gradient(105deg, 0.88 → 0 bei 90%)`
  - Mobile: vertikal `0.88 → 0.30 bei 100%`
- [x] **Schritt 4: Nachher-Messung + Kontrast-Wert mit konkreter Zahl**
  - Screenshot: `_logs/1.8-hero-after-a.png`
  - Befund: `_logs/1.8-hero-after-a.json`
  - **Subline-Kontrast: min 11.67:1, max 14.44:1 (alle 3 Sample-Punkte ≥ 11.67:1) → PASS WCAG AA (Schwelle 4.5:1)**
- [x] **Schritt 5: Option B nicht nötig** — Option A ergab bereits 11.67:1
