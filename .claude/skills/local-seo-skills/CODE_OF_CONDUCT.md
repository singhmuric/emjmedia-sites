# Code of Conduct

Local SEO Skills is built for the local SEO community — freelancers, consultants, agencies, in-house marketers. Everyone should be able to contribute, ask questions, and report issues without friction.

## Standards

We expect participants to:

- Be direct and specific. Feedback on code, skills, and docs should focus on the work, not the person.
- Assume good faith. Contributors come from many backgrounds and skill levels.
- Keep disagreements technical. When something is wrong, say what's wrong and why; propose a fix.
- Respect others' time. Search before asking; include reproduction steps in reports; keep PRs focused.

Behavior that gets someone removed from the project:

- Personal attacks, slurs, or threats — in issues, PRs, commits, Discord, or any project channel
- Publishing private information about others without permission
- Deliberate disruption of discussions or review processes
- Sustained pattern of bad-faith engagement after feedback

## Scope

This applies to the GitHub repository (issues, PRs, discussions, commit messages, wiki), the community Discord, and any other project-branded space.

## Enforcement

Maintainers review reported behavior and decide on a response. Responses range from a private warning to temporary suspension to a permanent ban, depending on severity and pattern.

Report concerns to [help@localseoskills.com](mailto:help@localseoskills.com). Reports are handled in confidence. When the report involves a maintainer, the other maintainer takes point.

## Attribution

This code of conduct is adapted from common open-source community norms, informed by the [Contributor Covenant](https://www.contributor-covenant.org/). It is intentionally shorter than the full Covenant — the goal is a policy that's read, not just linked.
